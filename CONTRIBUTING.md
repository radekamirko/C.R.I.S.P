# Contributing to CRISP

Thanks for wanting to improve CRISP. Here's how it works.

## What we welcome

- **Bug fixes** — something in a skill file that's wrong, inconsistent, or broken
- **Template improvements** — better elicitation moves, cleaner output structure, missing fields
- **New examples** — real CRISP engagements (sanitize client info before submitting)
- **Phase 5 (Prove)** — this phase is intentionally thin right now; contributions welcome

## What we don't want

- Generic prompt engineering additions that aren't tied to a CRISP phase
- Technology-specific templates (CRISP is stack-agnostic by design)
- Changes to the core methodology without discussion first — open an issue

## How to contribute

1. Fork the repo
2. Make your changes in a branch
3. Submit a PR with a clear description of what you changed and why
4. Reference any relevant issue

## Output convention

Blank templates live in `/templates/`. Filled project outputs live in `[project]/docs/`. Never add filled outputs to `/templates/`.

## Code of Conduct

Be direct, be useful, don't be a jerk.
