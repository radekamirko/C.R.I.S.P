# START HERE — CRISP

You have a brief. Before touching code, you need to define the problem, align on outcomes, and write a spec Claude Code can actually follow. That's what CRISP does.

Five phases. Each one feeds the next. Each one has an exit checklist — don't skip ahead.

---

## How to start

**Claude Code** — drop this repo into your project, then run:
```
/crisp-start
```

**Any Claude** — paste this exactly:
```
Read skills/phase1-clarify/SKILL.md and begin Phase C — Clarify. Don't explain the methodology. Don't summarize what you're about to do. Just ask me about the project.
```

Claude guides you through the rest.

---

## What happens next

| Phase | What it does |
|---|---|
| **C — Clarify** | Define the problem. Painkiller test. Buy vs build. Go/No-Go. |
| **R — Results** | Stakeholders. Baseline. Success metrics. |
| **I — Investigate** | Process mapping. User journeys. UX discovery. |
| **S — Spec** | Tech stack. Backlog. MVP prioritization. Sprint-ready AI Specs. |
| **P — Prove** | Did the needle move? Validate against Phase R. |

---

## Rules

- Blank templates live in `/templates/`. Filled outputs live in your project's `docs/`. Never overwrite blank templates.
- Check every exit checklist before moving to the next phase.
- Never hand a sprint to Claude Code without a locked AI Spec.
