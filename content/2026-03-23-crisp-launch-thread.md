**Tweet 3:**
CRISP is a 5-phase BA/PM methodology for AI implementors.

C — Clarify the problem (painkiller test, constraints, go/no-go)
R — Define results (baseline the current state, agree on success)
I — Investigate the process (map every step)
S — Spec it out (agent architecture, backlog, UX, CLAUDE.md)
P — Prove it worked

---

**Tweet 4:**
The Spec phase is where most AI builds actually fall apart.

So CRISP gives you:
→ A design system template with cognitive UX principles baked in
→ UX spec: sitemap → flows → screens
→ CLAUDE.md template (what Claude reads every session)
→ Agent SKILL.md template per agent
→ AI Spec template per sprint

No ambiguity handed to Claude Code.

**Tweet 5:**
There's one rule in CRISP:

"Never discuss technology before the problem is defined.
Never design a solution before the finish line is set.
Never hand a feature to Claude Code without an AI Spec."

That's it. Follow those three and you'll ship the right thing.
